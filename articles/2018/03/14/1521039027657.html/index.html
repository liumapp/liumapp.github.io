Hosts与Lantern的冲突

发表于 2018-03-14

在蓝灯官方论坛关于lantern启动失败的帖子，有这么一句：“错误的Hosts文件配置会导致问题产生，所以解决办法请删除/etc/hosts…”当时看到这句话内心一万只草泥马在奔腾，明明只需要修改一小处地方硬是要删整个文件。

前言

我自己测试的环境是在Mac下，Windows环境不能保证一定如此。

产生问题

用任何一款编辑器打开/etc/hosts，加上对0.0.0.0地址的解析，比如

0.0.0.0 account.jetbrains.com

接下来开启lantern，那么会发现：本来运行正常的蓝灯，隔一段时间便会失去与代理的连接，并且很难再重新恢复。

解决办法

只需要保证hosts文件内没有对0.0.0.0地址的解析即可

# hosts # lantern
